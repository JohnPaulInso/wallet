## File Naming Conventions

### JavaScript Modules
- `app-*.js` - Core application modules
- `*-logic.js` - Feature-specific logic modules
- `firebase-*.js` - Firebase-related modules
- `local-*.js` - Local-only features

### HTML Pages
- Lowercase with underscores: `account_setup.html`
- Or hyphens for multi-word: `edit-goal.html`

### Batch Scripts (Windows)
- Lowercase with hyphens: `fix-java-version.bat`
- Purpose-driven names: `auto-clean-gradle.bat`

### Style Files
- Match corresponding JS module: `bpi-scanner.css` â†” `bpi-scanner.js`

## Key Architecture Patterns

### Single-Page App (SPA)
- `index.html` is the main shell
- Other pages are separate for specific features (accounts, calendar, goals)
- Navigation managed via `nav-state.js`

### ES Module Structure
- All core logic in ES6 modules
- Import/export for clean dependencies
- Window globals used sparingly for cross-module communication

### Data Flow
1. **Auth**: `app-auth.js` â†’ Firebase Auth â†’ user session
2. **Data**: `app-data.js` â†’ Firestore â†’ local state (`window.allTxns`)
3. **UI**: `app-ui.js` â†’ DOM updates â†’ visual feedback

### State Management
- Local: `localStorage` for persistence
- Session: `sessionStorage` for temporary state
- Global: `window.*` for runtime state (e.g., `window.allTxns`, `window.currentAccount`)
- Cloud: Firestore for multi-device sync

### Modal/Overlay Pattern
- Inline HTML in `index.html`
- Toggle via display/opacity CSS
- Backdrop blur for depth
- Example: BPI scanner (`#bpi-scanner-overlay`)

## Important Notes

### Do Not Edit
- `/www/*` files - always edit source in root, then sync
- `node_modules/` - managed by npm
- Generated build files in `/android/app/build/`

### Always Sync After Changes
1. Edit source files in root
2. Run `npm run sync-www` or `npm run build`
3. Then `npx cap sync android` if native changes needed

### Java Version Critical
- All Android builds require Java 17
- Java 21 will cause compile failures
- Auto-fix runs on every build via triple-layer protection
