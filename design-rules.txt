1. PREMIUM TRANSACTION ROW / CARD SPACING
--------------------------------------------------------------------------------
Standard metrics used in transaction cards (.premium-txn / .txn-item):

- Card Row Wrapper:
  * Padding: 14px top/bottom, 18px left/right
  * Alignment: display: flex; align-items: center; justify-content: space-between;
  * Inner Gap: 14px (between icon, details, and amount)
  * Border: 1px solid #f1f5f9 (Neutral standard border)
  * Left border colors: None (removed as per design requirements)
  * Shadow: 0 4px 6px -1px rgba(0,0,0,0.03), 0 2px 4px -1px rgba(0,0,0,0.02)

- Synced Card highlights:
  * BG color: #f8fafc (Clean light gray tint)
  * Border color: #e2e8f0

- New/Pending Card highlights:
  * BG color: #fffdf5 (Soft warm yellow tint)
  * Border color: #fef08a (Soft yellow outline border)

- Icon Container Box (.icon-box / .bpi-txn-icon):
  * Dimensions: 44px width, 44px height
  * Border Radius: 12px
  * Alignment: display: flex; align-items: center; justify-content: center;
  * Icon Font Size: 20px
  * Flex behavior: flex-shrink: 0;

- Transaction Details (.txn-details / .bpi-txn-info):
  * Alignment: display: flex; flex-direction: column;
  * Flex behavior: flex: 1; min-width: 0;

- Action Buttons (.bpi-txn-btn):
  * Dimensions: 32px width, 32px height
  * Border Radius: 50% (fully circular check and reject buttons)
  * Inner Gap: 6px between buttons


2. OVERLAY & GESTURE SHEETS (.bpi-scanner-sheet)
--------------------------------------------------------------------------------
- Backdrop Overlay (.bpi-scanner-backdrop):
  * Background: rgba(15, 23, 42, 0.4) (darkened layer)
  * Blur styling: backdrop-filter: blur(12px) (frosted glass background)

- Pull-down Sheet Content (.bpi-scanner-content):
  * Height: 90vh (forces it to fill the entire space below the top offset)
  * Border Radius: 28px 28px 0px 0px (curved top corners only)
  * Gesture Trigger: Pull-down close enabled on .bpi-sheet-handle-area
  * Gesture Threshold: Closable if dragged down >120px


3. TYPOGRAPHY STANDARDS
--------------------------------------------------------------------------------
- Merchant/Title Label (.txn-merch / .bpi-txn-merchant):
  * Font Size: 13.5px
  * Font Weight: 700 (relaxed from 800)
  * Colors: #0f172a (Light Mode), #f8fafc (Dark Mode)
  * Transformation: text-transform: uppercase; (or standard casing for matching)

- Subtitle / Meta (Date and Category) (.txn-sub / .bpi-txn-meta):
  * Font Size: 9.5px
  * Font Weight: 600 (relaxed from 700)
  * Colors: #64748b (Light Mode)
  * Line spacing / formatting: white-space: nowrap; (prevents date and category wrapping)
  * Overflow rules: overflow: hidden; text-overflow: ellipsis;

- Transaction Amount (.txn-amount / .bpi-txn-amount):
  * Font Size: 14px
  * Font Weight: 700 (relaxed from 800)
  * Colors: #ef4444 (Negative/Expenses), #10b981 (Positive/Synced)

- Note Label (.txn-note / .bpi-txn-note):
  * Font Size: 11px (matching main app transaction layout notes)
  * Font Weight: 600 (relaxed from 700)
  * Style: text-transform: none; (supports Title Case / standard casing)
  * Margin Top: 3px

6. MODAL DESIGN SYSTEM
--------------------------------------------------------------------------------
All confirmation and interaction modals follow iOS-style center modal pattern:

- Modal Overlay (.modal-overlay, #confirmation-modal):
  * Background: rgba(15, 23, 42, 0.75) (Dark translucent backdrop)
  * Backdrop Filter: blur(8px) (Frosted glass effect)
  * Position: fixed; top: 0; left: 0; right: 0; bottom: 0;
  * Z-index: 100000 (Above BPI scanner at 99999)
  * Animation: fadeIn 0.2s ease

- Modal Container (.custom-modal, .confirmation-modal):
  * Background: #ffffff (Pure white - always light theme)
  * Border Radius: 20px (Soft rounded corners)
  * Max Width: 340px
  * Padding: 28px 24px 20px
  * Box Shadow: 0 20px 60px rgba(0, 0, 0, 0.3)
  * Animation: scaleIn 0.3s cubic-bezier(0.34, 1.56, 0.64, 1) (Bouncy entrance)

- Modal Title (.modal-title, #confirmation-modal-title):
  * Font Size: 18px
  * Font Weight: 700
  * Color: #0f172a (Dark slate - high contrast on white)
  * Margin Bottom: 4px
  * Letter Spacing: -0.3px
  * Text Align: center

- Modal Message (.modal-message, #confirmation-modal-message):
  * Font Size: 14px
  * Font Weight: 500
  * Color: #64748b (Slate 500 - readable gray)
  * Line Height: 1.5
  * Margin Bottom: 24px
  * Text Align: center

- Modal Button Container (.modal-actions):
  * Display: flex
  * Gap: 12px
  * Flex Direction: row
  * Justify Content: space-between

- Primary Action Button (.modal-btn-primary, OK button):
  * Background: #06b6d4 (Cyan brand color)
  * Color: #ffffff (White text for maximum contrast)
  * Font Size: 15px
  * Font Weight: 700
  * Padding: 13px 28px
  * Border Radius: 14px
  * Border: none
  * Transition: all 0.2s ease
  * Hover: transform: scale(1.02); background: #0891b2; box-shadow: 0 4px 12px rgba(6, 182, 212, 0.3)
  * Active: transform: scale(0.98)

- Secondary Action Button (.modal-btn-secondary, Cancel button):
  * Background: transparent
  * Color: #64748b (Muted gray)
  * Font Size: 15px
  * Font Weight: 700
  * Padding: 13px 28px
  * Border Radius: 14px
  * Border: 2px solid #cbd5e1 (Light slate border)
  * Transition: all 0.2s ease
  * Hover: background: rgba(203, 213, 225, 0.2); border-color: #94a3b8; color: #475569
  * Active: transform: scale(0.98)

- Dark Mode Behavior:

11. SHADOW SYSTEM
--------------------------------------------------------------------------------
Elevation hierarchy using box-shadow:

LEVEL 0 - Flat (No shadow):
- Use: Flush elements, backgrounds

LEVEL 1 - Subtle elevation:
- Shadow: 0 1px 2px rgba(0,0,0,0.04)
- Use: Card borders, subtle separation

LEVEL 2 - Low elevation:
- Shadow: 0 2px 4px rgba(0,0,0,0.06)
- Use: Buttons, small cards, hoverable items

LEVEL 3 - Standard elevation:
- Shadow: 0 4px 6px -1px rgba(0,0,0,0.03), 0 2px 4px -1px rgba(0,0,0,0.02)
- Use: Transaction cards, standard components

LEVEL 4 - Medium elevation:
- Shadow: 0 4px 12px rgba(0,0,0,0.08)
- Use: Dropdown menus, notification center

LEVEL 5 - High elevation:
- Shadow: 0 10px 25px rgba(0,0,0,0.1)
- Use: Notification dropdowns, context menus

LEVEL 6 - Dramatic elevation:
- Shadow: 0 20px 60px rgba(0,0,0,0.4)
- Use: Modals, overlays, maximum depth

SPECIAL SHADOWS:
- Card Active: 0 5px 15px rgba(0,0,0,0.3)
- BPI Card: 0 5px 15px rgba(92, 10, 10, 0.4)
- Notification Badge: 0 2px 4px rgba(239, 68, 68, 0.3)


12. ANIMATION & TRANSITION SYSTEM
--------------------------------------------------------------------------------
Consistent motion design for smooth UX:

TIMING FUNCTIONS:
- ease-in-out: Standard smooth (general transitions)
- cubic-bezier(0.4, 0, 0.2, 1): Material Design standard (cards, nav)
- cubic-bezier(0.34, 1.56, 0.64, 1): Bouncy (modals, buttons, special effects)
- cubic-bezier(0.16, 1, 0.3, 1): Smooth deceleration (loads, fades)

DURATION STANDARDS:
- 0.15s - Ultra fast (micro-interactions, button presses)
- 0.2s - Fast (button hovers, small state changes)
- 0.22s - Quick (tab transitions, quick fades)
- 0.28s - Standard (fades, subtle animations)
- 0.3s - Default (modal backgrounds, most transitions)
- 0.32s - Medium (slides, content entry)
- 0.4s - Slow (major transitions, header scroll, complex animations)
- 0.85s - Special (spinner rotations)
- 1.05s - Very slow (shimmer effects)
- 1.5s - Ambient (shimmer premium effect)
- 2s - Background (skeleton pulse)

NAMED ANIMATIONS:
- fadeIn: Opacity 0 → 1 (0.2s ease)
- scaleIn: Scale 0.9 → 1 with opacity (0.3s bouncy)
- slideUp: TranslateY(20px) → 0 with opacity (0.15s ease-out)
- shimmer-premium: Diagonal gradient sweep (1.5s infinite)
- skeleton-pulse: Background color pulse (2s infinite)
- bioPulse: Scale with shadow expansion (2s infinite)
- spinAnimation: 360° rotation (0.85s infinite linear)
- fadeInLoad: Opacity + translateY + scale (0.4s smooth deceleration)
- tabFadeInQuick: Quick opacity + translateY (0.22s)
- tabSlideInFromRight: Slide from right (0.32s smooth deceleration)
- tabSlideInFromLeft: Slide from left (0.32s smooth deceleration)

HOVER INTERACTIONS:
- Buttons: scale(1.02) on hover, scale(0.98) on active
- Cards: subtle background shift, no scale
- Icons: scale(1.05) on hover, scale(0.95) on active
- Notification bell: background appears, slight scale up

17. ACCESSIBILITY GUIDELINES
--------------------------------------------------------------------------------
Design considerations for usability:

TOUCH TARGETS:
- Minimum: 40x40px (notification icons, small buttons)
- Comfortable: 48x48px (category icons, profile buttons)
- Large: 56x56px (FABs, primary actions)

CONTRAST RATIOS:
- Body text: Minimum 4.5:1 (WCAG AA)
- Large text: Minimum 3:1 (WCAG AA)
- UI components: Minimum 3:1

TEXT READABILITY:
- Line height: 1.4-1.6 for body text
- Max line length: ~60-70 characters
- Letter spacing: Adjust for uppercase labels

FOCUS STATES:
- Visible focus indicators on interactive elements
- Keyboard navigation support (where applicable)

COLOR USAGE:
- Never rely solely on color to convey information
- Use icons + text + color together
- Sufficient contrast for color-blind users


18. STATE MANAGEMENT PATTERNS
--------------------------------------------------------------------------------
Visual feedback for different states:

LOADING STATE:
- Skeleton screens with shimmer animation
- Pulse animation for backgrounds
- Maintain layout structure
- Keep skeleton until data arrives

EMPTY STATE:
- Centered message with icon
- Helpful description text
- Primary action button to add content
- Muted color scheme

ERROR STATE:
- Red accent (#ef4444)
- Clear error message
- Retry action button
- Maintain context of what failed

SUCCESS STATE:
- Green accent (#10b981)
- Confirmation message (brief)
- Auto-dismiss or manual close
- Positive icon (checkmark)

DISABLED STATE:
- Reduced opacity (0.5-0.6)
- Muted colors (#cbd5e1)
- No hover effects
- Cursor: not-allowed

HOVER/ACTIVE STATE:
- Subtle scale transformations
- Background color shifts
- Shadow elevation changes
- Fast transitions (0.2s)


19. BEST PRACTICES & GUIDELINES
--------------------------------------------------------------------------------
Rules to maintain design consistency:

SPACING:
✓ Use the defined spacing scale (avoid arbitrary values)
✓ Maintain consistent gaps within similar components
✓ Increase spacing for visual hierarchy
✗ Never use spacing less than 2px (creates cramped UI)

TYPOGRAPHY:
✓ Use font weights from the system (600, 700, 800)
✓ Match font sizes to defined use cases
✓ Apply letter-spacing to uppercase text
✗ Avoid font weights 400 or 500 for primary UI text
✗ Never exceed 3 font sizes in a single component

COLORS:
✓ Use semantic colors (success = green, error = red)
✓ Maintain category color associations
✓ Check contrast ratios in both themes
✗ Never use pure black (#000) except for overlays
✗ Avoid creating new color variants without system update

ANIMATIONS:
✓ Use cubic-bezier for refined motion
✓ Keep durations under 0.4s for UI interactions
✓ Apply will-change for performance-critical animations
✗ Avoid long animations (>0.5s) on frequent interactions
✗ Never animate height/width directly (use transform)

MODALS & OVERLAYS:
✓ Center modals for confirmations
✓ Use bottom sheets for extensive content/forms
✓ Apply backdrop blur for depth
✓ Include clear dismiss actions
✗ Never stack more than 2 modal layers
✗ Avoid modals for non-critical information

BUTTONS:
✓ Primary action = filled with brand color
✓ Secondary action = outlined or ghost style
✓ Maintain 44px minimum touch target
✓ Use consistent padding (13px vertical minimum)
✗ Never use more than 2 primary buttons in same context
✗ Avoid button text over 20 characters
